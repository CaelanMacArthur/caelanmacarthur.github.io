Alexa Skills Development


Intents


Amazon Alexas intents are functions attempts to fulfill end user's queries.Intents can have positional arguments called slots that to map (sort of a like HashMaps, or Linked Lists) to improve the intents capabilities.


Utterances

Utterances are snip bits of end-user responses so that the skill can know how to respond.


Multimodal Responses


Multimodal Alexa Skill allows for Alexa skills in both audio and visual responses to user's requests.


Handlers and Alexa’s SDK


Handlers are used to communicate requests from Intents with Alexa Skills Development SDK. The intents intake the utterances from the end-users that see if any slots are mapped to that intent. Then take that input to the handlers and relay that to the Alexa Skills Development SDK.


Skill Types and Their API’s

There are many different skill types and each has a different API they call for that functionality.

1)

Smart Home Skill control devices in your home. This is done by connecting the Smart Home Skill APIs, Smart Home Energy Skills, Smart Home Security Skills, and Multi-capability Skills. Examples of objects that it controls and interacts with are cameras, lights, locks, thermostats, cameras, and security systems.


2) Flash Briefing Skills are skills that provide news content to users. The returned information is often brief like news headlines.

3) Automotive Skills are skills that are meant to integrate with the user's car. There are two different types of functionality in this.

a) Driving Skills allow users to be productive while driving so that they can keep an eye on the road.

b) Connected Vehicle Skills Alexa to control your car functionality like turning on your car.

4) Alexa Skills for Business, Hospitality, and Residential

a) Alexa for Business: Alexa business skills can deeply integrate with a business infrastructure to help the workload of the business by automating tasks. Like checking to see if there are any meeting rooms available and then scheduling that meeting with the booking API. Similarly, the conferencing skill allows employees to control the room lights, the volume of speakers, etc.

b) Alexa Hospitality uses technology to integrates to deploy and manage functionality in hotels, resorts, vacation rental communities, hospitals, etc. The Alexa Hospitality functionality is rather extensive for how it integrates. It uses the Property, Room Type, and Room API, Role API, Unit Settings API, Endpoint API, Device Group Management API, Discovery Session API, Skill Management API, Communications API, and Notifications API.

5) Entertainment Device Skills controls smart home entertainment and audiovisuals (AV) devices such as smart TVs, cable boxes, and other entertainment devices. The APIs it uses is Alexa.ChannelController, Alexa.EqualizerController, Alexa.InputController, Alexa.KeypadController, Alexa.Speaker, Alexa.StepSpeaker, and Alexa.WakeOnLANController.


6) Game Skills adds on visuals or you can create a new game with an integrated voice and on-screen experience. The Game Skills support web applications with Alexa Skill integration through the Game Skills API.


7) Knowledge Skill allows users to ask data queries from classical spreadsheets. It integrates with Alexa for Business, Alexa for Hospitality, or Alexa for Residential skills. It also works along with those application's API.


8) Alexa Skill Listssimply create lists or to-do lists for the end-user. It generally recorded a connect app of the user.


9)Music, Radi o, and Podcast Skills allow users to control and interface audio streaming (Spotify, Pandora, etc). The Alexa Music, Radio, and Podcast Skill API are used to control and request streaming audio for music.


10) Networking and Wi-Fi Skills allow Alexa to connect with other devices in an end-users home. This is done through Alexa.Networking.HomeNetworkController, Alexa.Networking.ConnectedDevice, and Alexa.Networking.AccessController APIs that connect all other devices.


11) Video Skills Video Skills allow video streaming to devices that have Alexa integration. For example, Amazon Fire TV. Video Skill API hands the requests when fetching video.


12) Custom skills are third-party Alexa skills developed that allow flexibility outside Amazon regular skills. The idea is to a variety is to provide the end-user of Alexa's as much diversity functionality as possible

Testing Third Party Skills with Honeytokens


In the planning phase of figuring out how to test with honeytokens, we need to figure out the variables when choosing data for testing. Data needs be able to easily tracked and data is something that should easily produced at a low cost. Since Amazon Alexa Ecosystem is so large there are so many data type values available.


Constraints During Testing


Since Alexa ask-alexa-core seemingly only takes strings for user requests and image types to display on the Echo Show screen. Any alternative formats of sending packets to Alexa not encoded to a string would potentially cause a runtime error if Alexa does not have error handling for data types besides strings. However, Alexa skills ask-sdk-runtime puts input into an array to store in memory, error handling for an unexpected input but seemingly does not for the size of the incoming string. As such it is nesscary to covert data formats that are not string into strings.


Data Conversion for Amazon Alexa


Since data types in from the intents in the Alexa SDK are only being sent as strings any file that is being handled by Alexa that is not a string type while need to be converted. This would be done switch files removing the file header, stripping the data of the file type, and leaving it in UTF-8. Then put it into a string variable to sent.


Logically any Alexa skill that is hosted by AWS would use Amazon Kinesis Data Streams streaming cloud solution. This means that Amazon seemingly would have more access to their backend to code to see how to use data is being handled.



There is a lot of third-party open-source streaming solutions. A third-party skill could simply take this solution implement it and store there the files in a database. Leaving for the data to be misused.
