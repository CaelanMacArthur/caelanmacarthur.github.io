Problems with Automation and to Address them

Currently, automation for testing Amazon Alexa with honeytokens is being stopped by reCaptcha.

However, reCaptcha is not impossible to get around, and there are a couple of common solutions. The easiest is to mimic a human and therefore not ever be prompted to solve reCaptcha. However, when using a program like Selenium you can come across problems with User-Agent. User-Agent is a request header that identifies to a server what OS, window size, and web browser you are using. You can use the time library to mimic human behavior when someone uses a website. Use fake-useragent to get past User-Agent. So fake-user agent gives User-Agent that information by that references databases containing User-Agent expects.

For example, if you are using chrome you use to do. This code is being referenced from here:

from fake_useragent import UserAgent

ua = UserAgent()

ua['google chrome']

It imports a UserAgent database and then passes it into a list so that Python can send it to the websites the appropriate HTTP headers to the receiving User-Agent.
