Voice Translator and Synthesizer

To interact with Alexa, we needed to create a program to speak and record Alexa’s responses.

Voice.py is the voice synthesizer I created to talk to Alexa. It has a function that is capable of speech recognition to translate Alexa recordings and a function that handles audio streams to record, and then have all of that recorded to a JSON file for data analysis for the final results.

To create the voice synthesizer functionality came in two steps. Firstly is a Python library called gTTS. gTTS stands for Google Text-to-Speech that calls Google Translate’s text-to-speech API. gTTS takes a bytestring supplied by the developer and turns them into .mp3 files. gTTS also offers the ability to switch the language being recorded by changing the language code. Additionally, in case the skill uses more of a region-specific accent that it recognizes better, gTTS also has localized accents you can choose to create further capability. Secondly, I used Playsound to have those files output to Alexa.

However, the biggest challenge is recording the audio itself. Recording needs an audio stream to be stable, compressing, and decompressing into a file properly. I need to know the length of the audio recording to understand the bit rate at which I need to record. The usual response is around 10 seconds so I set the 1024 audio samples, with two channels for how the data is read, and bit rate to 16. This allowed for a stable and scalable audio stream that could handle up to about 30 seconds if needed. To accomplish this, I used Pyaudio, a Python library that does handle the majority of the heavy lifting.

Lastly, SpeechRecognition is a Python library that translates audio files to text. Just like gTTS, you can change the language code to recognize different languages. However, one potential problem is that SpeechRecognition may not work with some language bundles.
