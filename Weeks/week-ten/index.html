Getting Oauth and Automation

Account linking allows Oauth to be implemented to confirm the identity of a user. It accomplishes this by linking to accounts that belong to the user.

For example, if I have a Google account and Amazon Alexa, the third party skill can allow me to sign in to my Google account during the beginning of the session to identify my identity. This is different than two-factor authentication (2FA), where it sends you an SMS.  

Account linking can be set up through Authenticator apps like Google Authenticator, or Microsoft  Authenticator. Additionally, it can be set up by calling an API and sending through a web application. Examples of available APIs include Google’s API, or logging through the Amazon web portal. That being said, third-party skills can use any Oauth service they want, including a custom-built solution they made themselves.

In SkillVet, they found a lot of skills have broken links and when I went over the data, I realized that a lot of skills that had account linking did not have any permissions listed. Meaning it is possible that the third-party skills are only listing the requested permissions at the beginning of setting up account linking, but are not listing any permissions needed after.

In speculation, it is also possible that during the account linking process with a third party skill may be able to get away with not showing the requested permissions at all, or use dark patterns to obfuscate them. This is only likely to happen if it is a custom Oauth solution and it is a web application.

Since so many of the third-party Alexa skills have account linking and no permissions listed, it is crucial to be able to test these skills for privacy violations. However, there are a lot of factors to work around when using automation and testing. You either need to enter get Oauth manually which is lengthy and not scalable, or automate.

However, automating testing and getting OAuth to work can be a little tricky. Third-party skills can use any OAuth service they want including a custom solution. Platforms that it can be conducted on include mobile or web.

In this case, the only way to automate testing is to create program consent of the following features:

  • Auto-clicker
  • Copy and paste
  • Mouse and Keyboard
  • Logic to determine if getting Oauth is successful

Selenium is not recommended given the variety of services (including custom solutions) available to third-party skills for Oauth.
